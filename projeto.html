Introdução ao Git e GitHub

O que é o Git?

O Git é um sistema de controle de versão distribuído, open-source e gratuito, criado por Linus Torvalds. Ele é projetado para lidar com tudo, desde pequenos a grandes projetos, com velocidade e eficiência. O Git permite que desenvolvedores trabalhem em um mesmo projeto simultaneamente, rastreando todas as alterações feitas nos arquivos ao longo do tempo. Isso facilita a colaboração, permite reverter para versões anteriores e gerenciar o histórico do código.

O que é o GitHub?

O GitHub é uma plataforma de hospedagem de código para controle de versão e colaboração, que usa o Git. Ele funciona como um "repositório" online, onde você pode armazenar seus projetos (repositórios Git) e colaborar com outras pessoas. O GitHub oferece funcionalidades sociais, como a capacidade de seguir outros usuários, visualizar seus projetos, e abrir "pull requests" para sugerir alterações no código de outras pessoas.

Diferenças entre Git e GitHub

É comum confundir os dois, mas a diferença é simples:

  • Git: É a ferramenta de software, o sistema de controle de versão que você instala e executa no seu computador. Ele gerencia as versões do seu código localmente.
  • GitHub: É a plataforma na web, o serviço de hospedagem que armazena seus repositórios remotamente. Ele usa o Git para funcionar, mas adiciona uma camada social e de colaboração sobre ele.

Principais Comandos Linux para usar o Git Bash

O Git Bash é um emulador de terminal que permite usar comandos Linux no Windows. Aqui estão alguns comandos básicos e essenciais:

  • ls: Lista os arquivos e diretórios no diretório atual.
  • cd <diretório>: Entra em um diretório específico. Por exemplo, cd Desktop.
  • pwd: Exibe o caminho do diretório atual.
  • mkdir <nome_do_diretório>: Cria um novo diretório.
  • rm -rf <nome_do_diretório>: Remove um diretório e todo o seu conteúdo.
  • touch <nome_do_arquivo>: Cria um arquivo vazio.

Principais Comandos Git para o repositório

A seguir, comandos comuns para interagir com o repositório caixetovisk/aula_git, como indicado no arquivo README.md:

  • git clone <URL>: Clona um repositório existente do GitHub para a sua máquina local. Ex: git clone https://github.com/caixetovisk/aula_git.git.
  • git add .: Adiciona todos os arquivos alterados ao "staging area", preparando-os para o próximo commit.
  • git commit -m "mensagem": Salva as alterações no histórico local do repositório, junto com uma mensagem descritiva.
  • git push origin main: Envia as alterações da sua máquina local para o repositório remoto (no GitHub).

Como criar um repositório no GitHub

  1. Acesse o GitHub e faça login.
  2. No canto superior direito, clique no sinal de + e selecione New repository.
  3. Dê um nome ao seu repositório. Opcionalmente, adicione uma descrição.
  4. Escolha se o repositório será Public (público) ou Private (privado).
  5. Clique em Create repository. Seu novo repositório está pronto!

Como ligar em um projeto existente na máquina

Se você já tem um projeto em seu computador e quer enviá-lo para um novo repositório no GitHub, siga estes passos:

                
cd /caminho/do/seu/projeto
git init
git add .
git commit -m "Primeiro commit"
git branch -M main
git remote add origin https://github.com/seu_usuario/nome_do_repositorio.git
git push -u origin main

Lembre-se de substituir seu_usuario e nome_do_repositorio pelos seus dados.

Como gerar uma versão com Git

Gerar uma nova versão significa criar um "commit", que é um instantâneo das alterações no seu projeto. Os passos são:

                
git add .
git commit -m "Descreva as alterações que você fez aqui"

É crucial escrever uma mensagem de commit clara para que você e outras pessoas entendam o que foi alterado.

Como enviar da máquina para o GitHub

Após gerar uma versão (commit) em sua máquina local, você precisa enviá-la para o repositório remoto no GitHub. Use o comando:

                
git push

O comando git push envia os commits do seu repositório local para o repositório remoto, sincronizando as versões.
